1. Using Questr — the end-user journey

Three activities: Explore any place on Earth, Play a timed hunt on the real landmarks around it, and Remember the day as a collage journal page. Found landmarks flow into the Journal automatically.

2. Backend — how the components are wired

The SPA is fully static (Cloudflare Pages, no secrets). Everything stateful lives on the edges: free geo APIs for exploration, one small always-on box for collage builds, S3 for images, and a GitHub Action that drives the same engine from a markdown file.

3. Collage — two ways in, one evaluator/optimizer loop

Both entry paths converge on the same generate → judge → refine loop. The judge is the vision model when a key is present (scoring by template resemblance on the GitHub path), and a deterministic composition heuristic offline — the loop always completes.
